## Approach

Output-neutral WORK and MEMCOST reduction in the Context Tree Weighting model. Until the node store hits its cap, a context's present nodes form a prefix {0..=frontier} of the depth axis (a node can only have been inserted if all its suffix nodes were), so predict now probes the path top-down only until the first absent depth; the deeper tail is provably empty and contributes pred=0.5 with no map probes. On high-entropy data this roughly halves CTW's per-bit HashMap probes, cutting both executed operators (WORK) and cache traffic (MEMCOST). Once an insert is refused the prefix invariant can break (a parent may be absent while a child exists), so a 'capped' flag flips predict back to the full leaf->root walk, keeping output byte-identical to the unskipped recursion on every input. Path keys are also stashed in predict and reused by update. Byte-identical on all six corpus files (SCORE 572423) incl. the cap-hitting binary_mozilla/source_samba, and all 9 round-trip tests pass.
